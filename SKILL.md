## 📖 完整示例：都市重生废柴逆袭

**用户输入**：
```
题材：都市重生
类型：废柴逆袭  
金手指：未来信息/商业头脑
目标平台：番茄小说
```

**AI输出**：

### 1. 书名（10个）
- 《重生2008：从摆摊到首富》- 数字式+时代标签
- 《重生1998：我靠预知未来赢麻了》- 重生+金手指明示
- 《被老婆骗光家产后，我重生了》- 痛点+反转

### 2. 简介
> 陈浩重生回到2008年，老婆还没卷钱跑路，弟弟还没把他害死。
> 这次他带着未来20年的商业记忆回来。
> 什么？2008年房价要涨？买！
> 什么？某科技公司要上市？梭哈！
> 从摆地摊开始，陈浩用二十年后的眼光，在2008年掀起了一场财富风暴。
> 只是没想到，重生回来的，不止他一个……

### 3. 人设卡
```
【主角】陈浩
- 身份：重生者，2028年意外死亡后回到2008年
- 处境：28岁，普通的哥，被老婆戴绿帽、被弟弟坑害
- 金手指：未来记忆，知道2008-2028年所有商业机遇
- 性格：谨慎、果断、有仇必报
- 目标：不再重蹈覆辙，活出不一样的人生

【金手指】未来信息库
- 表现：能清晰记得未来20年的大事件、股价、房价
- 限制：无法改变他人的自由意志
- 成长：从商业→科技→更大的舞台

【首个对手】陈志强（弟弟）
- 表面：好弟弟，关心哥哥
- 实际：觊觎哥哥的拆迁款，暗中与嫂嫂勾结
```

### 4. 世界观（简化版）
```
- 时代：2008年，龙国
- 地点：海城（三线城市）
- 特殊元素：重生者（除主角外，还有谁？）
- 力量体系：无异能，纯粹的商战
- 主要势力：陈家、商业对手、未来知己
```

### 5. 前30章大纲（核心情节点）
```
第1-3章：重生觉醒
- 重生醒来，发现回到2008年
- 发现老婆与弟弟偷情
- 获得金手指，开始布局

第4-10章：初战告捷
- 用未来信息赚取第一桶金
- 买下即将拆迁的地段
- 首次打脸弟弟

第11-20章：势力扩张
- 进入房地产市场
- 结识未来商业大佬
- 扩大打脸范围

第21-30章：第一个大高潮
- 发现更大的阴谋
- 与幕后势力对决
- 首个女主出现
```

### 6. 前3章开篇（首章节选）
> 陈浩睁开眼睛，看到的是斑驳的天花板，和窗外嘈杂的街道。
> "这是……2008年？"
> 他猛地坐起身，看向床头的日历——2008年3月15日。
> 上辈子的记忆如潮水般涌来：被弟弟骗光家产、老婆卷钱跑路、最后孤独地死在出租屋里。
> 这一次，他不会再犯同样的错误。

## 创作模式选择

本Skill提供两种创作模式，请根据您的需求选择：

### 🚀 模式A：AI全自动模式（推荐新手）
**适用场景**：没有具体创作想法，想要快速开始；第一次使用，希望快速上手

**使用流程**：
1. 输入一句话需求（如"写一个修仙爽文"或"都市逆袭爽文"）
2. AI自动生成完整的五阶段设定框架
3. 用户确认或微调后，直接进入内容生成

### 📝 模式B：渐进询问模式（适合有想法）
**适用场景**：有具体的创作想法；想要精细控制每个设定

**使用流程**：
1. 逐一回答12个问题（分五阶段）
2. 每个问题后可选择「由AI自动生成剩余所有设定」
3. 随时可以切换到全自动模式

## 核心原理速览

### 黄金五章法则
```
第1章：穿越/重生 → 世界观 → 身份处境（第3章前必须出现威胁或机会）
第2章：金手指暗示 → 冲突引入 → 读者期待建立
第3章：首战或挫折 → 能力展示 → 爽点初现
第4章：势力扩张 → 矛盾升级 → 期待强化
第5章：小高潮 → 收尾 → 新悬念
```

### 三翻四震结构
```
第一翻：反派嚣张 → 读者压抑
第一震：主角处境更惨 → 情绪加剧
第二翻：主角隐忍 → 读者期待
第二震：反派得寸进尺 → 愤怒积压
第三翻：主角反击 → 情绪引爆
第三震+第四震：连番打脸 → 情绪宣泄 → 满足达成
```

### 爽文核心公式
```
爽 = 压抑的积累 + 爆发的释放 + 预期的达成 + 超越预期的惊喜
```

### 字数控制速查表

| 创作阶段 | 建议字数 | 要点 |
|----------|----------|------|
| 黄金开篇（前三章） | 每章2000-3000字 | 第1章300字内出冲突 |
| 单章正文 | 2000-3000字 | 四段式结构 |
| 章节高潮 | 500-800字 | 全章最燃点 |
| 爽点单元 | 每5-10章1个大爽点 | 张弛有度 |
| 百万字规划 | 100万字≈350章 | 每50章一个剧情高潮 |

## 使用示例

### 示例1：生成爆款书名
**用户需求**：都市重生题材，情绪是爽文逆袭

**执行流程**：
1. 读取 `references/guides/title-guide.md`
2. 应用至少10种命名技巧
3. 生成30个不同风格的书名

**部分输出**：
- 《重生1990：从摆摊首富开始》（数字式+时代标签）
- 《被豪门赶出家门，我转身身家百亿》（反差式+爽点）
- 《重生那年，我先拿下了千亿女友》（悬念式+冲突）

### 示例2：卡文急救
**场景**：用户写到第50章，主角刚打败反派，不知道下一步剧情

**诊断**：剧情断崖型卡文

**急救方案**：
1. 新势力介入（建议3个具体方案）
2. 内部危机（主角阵营出现叛徒/资源危机）
3. 意外发现（发现新线索/新地图/新秘密）

### 示例3：爽点设计
**场景**：设计一个"扮猪吃虎"的高光时刻

**执行流程**：
1. 铺垫：反派嚣张、轻视主角
2. 压制：主角故意示弱
3. 爆发：关键时刻亮出底牌
4. 打脸：反派震惊、众人改观
5. 余味：情绪延续、期待下一轮

## 注意事项

1. **智能体主导**：所有创作内容由智能体基于用户需求和参考文档生成
2. **质量优先**：生成的内容应具备实际可用性，而非简单堆砌
3. **平台适配**：文案生成时必须严格按照目标平台的用户画像和风格规范调整
4. **创作自由度**：在参考文档基础上，鼓励创新和差异化
5. **迭代优化**：用户反馈后可快速调整，提供多个版本供选择
6. **写作质量**：避免模板化词汇，用动作/场景代替解释性叙述
7. **字数把控**：遵循字数分配指南，保持节奏张弛有度
