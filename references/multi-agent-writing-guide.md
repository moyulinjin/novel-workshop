# Multi-Agent 写作系统设计指南

> **来源**：Novel-OS + novel-multi-agents 核心架构整合
> 
> **目标**：从"单Agent指导"进化到"多Agent协作"

---

## 一、核心理念

### 传统单Agent的局限
```
用户 → 单Agent → 输出
问题：
- 所有任务都由一个Agent处理
- 上下文容易稀释
- 质量控制依赖单一判断
- 无法并行处理不同任务
```

### Multi-Agent协作优势
```
用户 → 协调Agent → ┬→ 规划Agent → 大纲
               ├→ 角色Agent → 人设
               ├→ 写作Agent → 正文
               └→ 审核Agent → 质量控制
```

---

## 二、五级编辑流水线

### Level 1：头脑风暴层（规划Agent主导）
**任务**：生成故事种子和概念

**输出**：
- 故事核心概念（1-2句话）
- 核心冲突定义
- 目标读者群体
- 题材类型判定

### Level 2：大纲规划层
**任务**：生成完整故事架构

**输出**：
- 主线大纲（起承转合）
- 支线设定
- 核心伏笔列表
- 分卷/分章节规划

### Level 3：草稿生成层（写作Agent主导）
**任务**：生成章节正文

**输出**：
- 章节正文（初稿）
- 章内情节点清单
- 需要审核关注的问题点

**上下文管理**：
```
草稿生成时自动携带：
- 风格指南（写作习惯）
- 角色设定（人物档案）
- 章节大纲（本章要求）
- 前文摘要（最近3章梗概）
```

### Level 4：修订优化层（审核Agent主导）
**任务**：质量检查和修订

**审核维度**：
| 维度 | 检查项 | 处理方式 |
|-----|-------|---------|
| 剧情逻辑 | 矛盾、漏洞、不合理 | 标记并给出修改建议 |
| 人物一致 | 性格偏离、行为矛盾 | 标注重述要点 |
| 爽点密度 | 是否达标 | 标注可增强位置 |
| 节奏把控 | 过快/过慢 | 调整建议 |
| 风格一致 | 是否符合你的风格 | 对比分析 |

### Level 5：定稿输出层
**任务**：最终整理和质量确认

**输出**：
- 符合发布标准的正文
- 创作笔记（方便后续续写）
- 伏笔追踪记录
- 下章待处理事项

## 三、Agent协作机制

### 任务交接协议
```
每个Agent完成工作后，输出标准格式：

【交接信息】
已完成：[具体内容]
待传递：[给下一个Agent的关键信息]
风险点：[需要关注的问题]
置信度：[本模块质量自评 1-10]

【下一个Agent的启动提示】
请基于以上信息，继续[下一步任务]
```

### 状态共享机制
```
共享状态文件：
├── story_state.md      # 故事当前状态
├── character_state.md  # 人物状态快照
├── plot_progess.md    # 情节点进度
├── hooks_tracker.md   # 伏笔追踪表
└── chapter_summary.md # 章节摘要（最近5章）
```

---

## 四、质量飞轮机制

### 循环迭代流程
```
初稿生成 → 质量评估 → 反馈修正 → 再评估
    ↑                                      ↓
    └──────────── 达标则进入下一章 ←───────┘

评估标准：
- 爽点密度 ≥ 1个/2000字
- 逻辑矛盾 = 0
- 人物偏离 = 0
- 节奏评分 ≥ 7/10
```

---

## 五、实战应用

### 调用Multi-Agent模式
```
当用户提供：
- "帮我写一整章"
- "续写这个故事"
- "完整的大纲规划"

自动启用Multi-Agent协作模式：
1. 先确认是否需要完整流水线
2. 按需启用对应层级的Agent
3. 逐步推进，用户可中途介入
```
