# 递归规划写作框架指南

> **来源**：WriteHERE - Heterogeneous Recursive Planning for Long-form Writing
> 
> **论文**：EMNLP 2025 Oral Presentation
> 
> **目标**：实现人类般的自适应写作能力

---

## 一、核心概念

### 什么是异质递归规划（Heterogeneous Recursive Planning）

传统写作框架的局限：
```
预定义工作流 → 固定大纲 → 顺序写作
问题：
- 写作过程中无法动态调整
- 发现新元素无法融入
- 遇到障碍只能硬着头皮写
```

递归规划的优势：
```
写作目标 → 递归分解 → 自适应执行 → 动态调整
特点：
- 边写边规划
- 发现新元素实时整合
- 遇到问题灵活绕行
```

## 二、三种认知任务类型

### T1：检索任务（Retrieval）
**定义**：获取外部信息或内部记忆

**场景**：
- 搜索相关背景知识
- 回忆前文设定
- 查找人物档案
- 获取世界观信息

### T2：推理任务（Reasoning）
**定义**：分析、推理、逻辑整合

**场景**：
- 情节逻辑推演
- 人物动机分析
- 伏笔合理性验证
- 多线索关联整合

### T3：创作任务（Composition）
**定义**：生成文本内容

**场景**：
- 场景描写
- 对话写作
- 情节叙述
- 情感表达

## 三、递归分解机制

### 分解终止条件

```
递归分解的停止条件：
1. 任务类型为T3（创作）且足够小（单一场景/段落）
2. 子任务可以立即执行
3. 不需要进一步分解

继续分解的条件：
- 子任务类型为T1（检索）：可能需要获取更多信息
- 子任务类型为T2（推理）：可能需要更深入分析
```

### 分解示例

```
任务：写一个"主角发现反派阴谋"的高潮场景

Level 1 分解：
├── T1: 检索前文铺垫的线索
├── T2: 推理阴谋的完整链条
└── T3: 创作发现阴谋的瞬间

Level 2 分解（T1分支）：
├── T1.1: 检索第一幕的伏笔
├── T1.2: 检索第二幕的暗示
└── T1.3: 检索第三幕的隐喻

Level 2 分解（T2分支）：
├── T2.1: 推理反派的目的
├── T2.2: 推理涉及的受害者
└── T2.3: 推理主角为何现在才发现

Level 2 分解（T3分支）：
├── T3.1: 创作震惊的表情描写
├── T3.2: 创作内心的独白
└── T3.3: 创作随后的行动
```

## 四、有向无环图（DAG）表示

### 任务依赖关系图

```
写作任务分解为DAG结构：

    [检索背景] ──┬──→ [推理真相] ──→ [创作场景]
                │           ↑
    [检索人物] ─┘           │
    ↑                      │
    └─────── [推理动机] ←───┘

节点状态：
- 激活（Active）：正在执行
- 挂起（Suspended）：等待前置任务完成
- 静默（Dormant）：暂不执行
- 完成（Completed）：已输出结果
```

### 状态化层次调度算法

```
执行流程：
1. 初始化所有节点为"挂起"
2. 找到入度为0的节点，设为"激活"
3. 执行激活节点
4. 更新依赖节点的入度
5. 重复2-4直到所有节点完成

动态调整：
- 如果执行中发现新需求，可插入新节点
- 如果某节点失败，尝试替代方案
- 用户可干预任何节点的状态
```

## 五、自适应写作流程

### 传统方式 vs 递归规划

**传统方式**：
```
写第一章 → 写第二章 → ... → 写到问题章节
         ↓
      遇到问题，无法回头修改整体结构
```

**递归规划方式**：
```
     ┌──── 推理：这里需要转折
     ↓
写第一章 → 发现需要伏笔 ──→ 暂停创作
                    ↓
              补充伏笔场景
                    ↓
              继续主流程
```

## 六、实战技巧

### 分解粒度控制

```
太粗：无法指导具体写作
例子："写第一章" → 太泛

太细：分解过于复杂
例子："写第一章第一句的动词" → 太碎

合适的粒度：
"写主角进入房间的场景，包含：
- 环境描写（3-5句）
- 主角反应（1-2句）
- 第一个发现（1个关键物品）"
```
