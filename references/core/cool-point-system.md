## 二、爽点公式

### 基础公式
```
爽点 = 铺垫(压抑) + 爆发(释放) + 反差(超出预期)
```

### 进阶公式
```
大爽点 = 小爽点 × 铺垫倍数 × 反差系数
```

**铺垫倍数**：压抑越久，释放越爽
**反差系数**：反差越大，爽感越强

## 三、爽点设计原则

### 3.1 铺垫要足
- 反派嚣张要写到位
- 主角压抑要写充分
- 读者愤怒要积压够

### 3.2 爆发要狠
- 反击动作干脆利落
- 冲击力要强
- 不要拖泥带水

### 3.3 反应要真
- 配角的震惊要真实
- 反派的绝望要可信
- 观众的反应要符合人设

### 3.4 余味要足
- 爽点后要有回味
- 可以短暂庆祝
- 为下一轮期待做铺垫

## 四、爽点密度规划

### 短篇（10万字以内）
```
每3章1个小爽点
每10章1个中爽点
结尾1个大高潮
```

### 中篇（30-50万字）
```
每5章1个小爽点
每15章1个中爽点
每50章1个大高潮
```

### 长篇（100万字以上）
```
每5-8章1个小爽点
每20-30章1个中爽点
每100章1个大高潮
卷末设小高潮
全书结尾设终极高潮
```

## 五、爽点场景创作模板

### 模板1：扮猪吃虎
```
【铺垫阶段】
1. 反派出场，轻视主角
2. 反派嚣张言论
3. 配角担忧/嘲讽
4. 主角平静回应（暗示会赢）

【压制阶段】
1. 反派动手，主角"挨打"
2. 众人觉得主角必输
3. 反派得意忘形

【爆发阶段】
1. 主角突然反击
2. 反派措手不及
3. 主角亮出底牌/真实实力
4. 反派震惊

【打脸阶段】
1. 反派被打
2. 众人震惊
3. 配角惊叹
4. 反派认怂/求饶
```

### 模板2：越级碾压
```
【开局】
1. 介绍双方实力差距
2. 所有人都觉得主角会输
3. 反派嚣张

【战斗过程】
1. 主角看似艰难应对
2. 众人嘲笑
3. 主角找到破绽

【反转时刻】
1. 主角爆发
2. 出乎意料的攻击方式
3. 反派无法抵挡

【结果】
1. 越级击败对手
2. 众人哗然
3. 主角淡然
```

### 模板3：身份揭露
```
【铺垫】
1. 主角被嘲讽身份低微
2. 有身份的人轻视主角
3. 主角被要求证明

【冲突】
1. 主角身份证据出现
2. 反派开始慌张
3. 众人将信将疑

【揭露】
1. 身份确认
2. 反派崩溃
3. 众人震惊

【收尾】
1. 主角被重新认识
2. 曾经轻视的人态度转变
3. 主角获得新地位
```

## 六、爽点组合技巧

### 组合类型

**身份爽 + 能力爽**
```
揭露身份的同时展现能力
如：皇子身份揭露 + 展现出天才实力
```

**情感爽 + 资源爽**
```
守护重要之人的同时获得资源
如：保护女主 + 获得女主家族支持
```

**复仇爽 + 打脸爽**
```
报仇的同时让对方颜面尽失
如：打败反派 + 让他当众出丑
```

### 叠加效应
```
单爽点 = 1x爽感
双爽点叠加 = 2.5x爽感
三爽点叠加 = 5x爽感
```

## 七、爽点雷区

### 避免情况

1. **铺垫不足**
   - 反派没嚣张就打败了
   - 读者还没压抑就释放了

2. **爆发太弱**
   - 反击不干脆
   - 打脸不够狠

3. **反应失真**
   - 配角反应平淡
   - 反派秒怂没有挣扎

4. **节奏失控**
   - 爽点太密集（麻木）
   - 爽点太稀疏（无聊）

5. **逻辑不通**
   - 主角突然变强没有理由
   - 反派降智为了被打脸
